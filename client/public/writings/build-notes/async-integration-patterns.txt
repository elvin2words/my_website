# Async Integration Patterns for Content-Driven UIs

Network variability is normal. The interface has to be robust by design.

Working patterns:

- loading and empty states are explicit
- fallback content paths are documented
- refresh paths are intentional, not accidental

Next version will add retry policy and stale cache behavior examples.
